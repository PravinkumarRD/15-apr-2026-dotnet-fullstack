How to create first Angular application - 

Step - 1 - Install Angular CLI globally on you machine using NPM
		npm install -g @angular/cli
Step - 2 - Check CLI installation and its version along with commands
		ng v
		ng --help
Step - 3 - Create new workspace for Angular

		ng new first-ng-app [Angular Workspace Name]
			Choose CSS as default stylesheet
			SSR/SSG - No
			AI tool - None

		cd first-ng-app

		ng serve or ng serve -o

	This will build your Angular application and run it on development web server which you can browser on PORT : 4200

Communication Between Parent and Child Component - 

1) Paren to Child - using property binding. Use @Input() metadata to decorate child component property which you can use as an attribute of child component selector. This attribute then can access parent component properties data

2) Child to Parent - 
	a) Declare the custom event
	b) Raise the event by emitting the data
	c) Decorate custom event with @Output() metadata
	d) Call parent function with custom event binding syntax in your child component selector where parent will receive the data sent by child component as a payload

To declare custom event, @angular/core package offers EventEmitter<T>
	
	@Output() sendMessage:EventEmitter<string> = new EventEmitter<string>();

	How does component get executed internally in angular framework?

	1) Angular creates an instance of your component
	2) Execute number of Lifecycle Hooks during the component execution process

Single Page Application [SPA] Development

	1) You will render the default webpage of your application along with the required contents in form of components, directives, pipes, services, models, ...
	2) Rest of the contents are loaded on demand [Lazy Loading] based on user's requirements


	@angular/router [contains the unique address for all the resources which you want to load eagerly/lazily]
	It also helps to maintain the browser history

	What is offered by Angular Router Library? [@angular/router]

	1) Route declaration 
		a) Default Route [http://localhost:4200] [Renders Default Component - EcomHome]
		b) Named Route [http://localhost:4200/home] [Renders Default Component - EcomHome]
		c) Named Route [http://localhost:4200/categories] [Renders Default Component - CategoriesList]
		d) Named Route [http://localhost:4200/products] [Renders Default Component - ProductsList]
		e) Parameterized/Dynamic Route [http://localhost:4200/products/3782] [Renders Default Component - ProductDetails]
	2) Router Outlet - Component
		<router-outlet></router-outlet>
	3) It offers declarative route attributes
	4) It also offers programmatic redirection support
	5) It offers route protection [Route Guards]
	6) Pass data with routes [Parameters/Query String]
	7) Eager Loading and Lazy loading of components
	8) It offers services to read URLs and their parts [ActivatedRoute]

class UserForm{

	uForm:FormGroup = new FormGroup({
		username: new FormControl(state, Validators.required),
		email: new FormControl(state,[Validators.required, Validators.Email,..]),
		name:: new FormControl(state, Validators.required),
		address: new FormGroup({
			zipcode:new FormControl(state,[Validators.required, Validators.minlength(6), Validators.maxlength(6)],
			geo:new FormGroup({
				lat:new FormControl(),
				lng:new FormControl()
			})
		})
	})
}
