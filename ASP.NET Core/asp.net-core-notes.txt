Introduction to Entity Framework Core [EF Core]

Entity Framework Core (EF Core) is Microsoft’s modern, lightweight, cross‑platform Object‑Relational Mapper (ORM) that lets .NET developers work with databases using C# objects instead of raw SQL. It simplifies data access, supports multiple database providers, and is widely used in enterprise, cloud‑native, and layered architectures

Key Features of EF Core
1) Cross‑platform: Works on Windows, Linux, and macOS.

2) Lightweight & Extensible: Designed to be modular, with support for plugins and customizations.

3) Database Provider Support: Works with SQL Server, PostgreSQL, MySQL, SQLite, Oracle, Cosmos DB, and more.

4) LINQ Integration: Query databases using strongly typed LINQ expressions instead of SQL.

5) Change Tracking: Automatically tracks modifications to objects and updates the database efficiently.

6) Migrations: Provides a way to evolve database schema alongside application code.

7) Performance Optimizations: Includes compiled queries, batching, and caching for speed.


Core Concepts
Model
Defined using entity classes (C# classes mapped to tables).
Relationships (one‑to‑many, many‑to‑many) are expressed via navigation properties.
DbContext
Represents a session with the database.
Provides APIs for querying (DbSet<T>) and saving data.

Example - 
	public class AppDbContext : DbContext
	{
	    public DbSet<Customer> Customers { get; set; }
	}

Querying
Use LINQ to query:
Example
	var customers = context.Customers
           .Where(c => c.City == "Pune")
           .ToList();

One-To-One

1 User will have 1 Profile

class User {
	[Key]
	public int UserId {get; set;}
	public string Email {get;set}=string.Empty;
	public Profile Profile {get;set;}//Navigation Property

	public ICollection<Order> Orders{get; set;}
}


class Profile{
	[Key]
	public int ProfileId {get; set;}

	public int UserId {get; set;}
	public User User {get; set;}
	
}

//One-To-Many

class Order{
	[Key]
	public int OrderId {get; set;}

	public int UserId {get; set;}
	public User User {get; set;}

}


//May-To-Many
class Store{
	[Key]
	public int StoreId {get; set;}
	public ICollection<Product> Products{get; set;}	

}

class Product{
	[Key]
	public int ProductId {get; set;}
	public ICollection<Store> Stores{get; set;}

}
