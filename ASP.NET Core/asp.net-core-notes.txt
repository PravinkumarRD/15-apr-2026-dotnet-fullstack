Why MVC?
1) It is an architectural style implementation
2) Loose coupling - Separation of Concern 
3) Supports TDD [Test Driven Development]
4) Easy testing
5) Conditional Rendering
6) Precise control over HTML
7) Complete control over Application Request Lifecycle [ASP.NET MVC Core]
8) In MVC, 
	i) Controller can work alone
	ii) Controller can work only with model
	iii) Controller can work only with View
	iv) Controller can work with View and Model
9) Reusability

ASP.NET Core - 
	1) State management
	2) Custom Middleware configuration
	3) Authentication and Authorization
	4) Caching [Data and Pages]
	5) Partial View support
	6) Data Access [ADO.NET (old), EF Core - ORM]
	7) Global Error Handling
	8) CORS configuration
	...



ASP.NET Core MVC Application Request Lifecycle [Collection of Middleware from where your request will go through]

ASP.NET offers number of middleware out-of-the-box and you can also write your own middleware called custom middleware

Begin Request Application Lifecycle


Middleware => Function => (Request, Response, Next)


Routes [A unique path to a specific resource]. Every web application contain collection of routes

What is a controller?
It is a special class which contain actions methods. The action method is the ultimate destination for your request where your request will be processed and the response will be generated

Route Address/Path
	https://localhost:8937/{controller=Home}/{action-method=Index}/{id?}
	
	[Default Route] https://localhost:8937 [With Default Controller and Default Action Method]
       [Specific Route] https://localhost:8937/Employees [Employees controller with default action method]
       [Specific Route] https://localhost:8937/Employees/List
[Dynamic Route]	https://localhost:8937/Employees/Details/1020
[Dynamic Route]	https://localhost:8937/Employees/Details?id=1020


Request => Routes Collection => Controller => Action Method

By default, ASP.NET Web Application runs on Kestrel Web Server
