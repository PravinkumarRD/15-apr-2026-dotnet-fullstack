What is Typescript?
	1) It is a superset of JavaScript
	2) It is a strongly typed language
	3) It compiles the code into plain JavaScript for a given ECMAScript Version
	4) Developer Language

TSC.exe [Compiler of Typescript] It transpiles the code of Typescript

	Transform + Compile = Converting/Transforming Typescript code into JavaScript

	Typescript => TSC.exe => JavaScript

	Typescript compiler is configurable [tsconfig.json/command line arguments]

Official Site - https://www.typescriptlang.org/

Why Typescript [30+ question for answering this question]

1) It has new types
2) Support strong typing along with Type inference
3) Supports OOP features
4) Interface support
5) Generics
6) Meta programming


Step to get typescript on your machine - 

	1) Install Node JS LTS version which will come with NPM [Node Package Manager]
	2) Run below commands - 
		node -v
		npm -v
	3) Install Typescript globally on your machine
		npm install -g typescript
		typescript -v

In ES6 [2015], we can create class with class keyword. But it still does not support access modifies in ECMAScript classes.

	Private members are declared using # symbol and these private members are declared using _ notation. For example #_socialId;
	Public members are without #. For example, employeeId; [This is a public members]. Constructor is written using constructor keyword
	For example - 
		class Employee{
			constructor(fname,lname,city){
				this.firstName=fname;
				this.lastName=lname;
				this.city=city;
			}
			firstName;
			lastName;
			city;
			printEmployeeInfo(){
				return `Employee ${this.firstName} ${this.lastName} lives in city ${this.city} !`;
			}
		}

How ESM works?

1) Resolve the dependencies
2) Load the dependencies [Modules are loaded exactly once]
3) Execute the dependencies


Main => Basic-Math => Advance-Math [Module Resolution] 
Advance Math => Basic Math => Main [Module Loading and Execution]
